# ImportPoly - AutoCAD 坐标导入工具

## 功能描述

`ImportPoly` 是一个 AutoLISP 脚本，用于将 CSV 格式的坐标数据导入 AutoCAD，并自动生成多段线（Polyline）或 3D 多段线（3D Polyline）。

## 功能特性

- ✅ 支持 CSV 格式坐标文件导入
- ✅ 三种 Z 轴处理模式可选
- ✅ 自动跳过表头行
- ✅ 坐标数据验证
- ✅ 系统变量自动保存/恢复
- ✅ 错误提示和用户反馈

## 使用方法

### 1. 加载脚本

在 AutoCAD 命令行执行：
```
(load "ImportPoly.lsp")
```

或者通过菜单：`工具` → `加载应用程序` → 选择脚本文件

### 2. 执行命令

在命令行输入：
```
ImportPoly
```

### 3. 按提示操作

#### 步骤 1：选择导入模式
```
选择导入方式: [1-保持原始3D | 2-压平至Z=0 | 3-统一高度] <1>:
```

**模式说明：**

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **1** | 保持原始 3D 坐标 | 地形图、3D 建筑模型 |
| **2** | 压平至 Z=0（平面） | 平面图纸、2D 设计 |
| **3** | 统一高度值 | 需要指定统一高程的场景 |

#### 步骤 2：若选择模式 3，输入统一高度
```
输入统一的Z高度值:
```

#### 步骤 3：选择 CSV 文件

弹出文件对话框，选择包含坐标数据的 CSV 文件。

#### 步骤 4：自动生成多段线

脚本将自动生成多段线并显示成功信息：
```
成功绘制多段线，共 XXX 个节点。
```

## CSV 文件格式

### 必需格式

**第一行：表头（必需，会自动跳过）**
```csv
ID,X,Y,Z
```

**后续行：坐标数据**
```csv
1,100.5,200.3,50.0
2,110.5,210.3,51.2
3,120.5,220.3,52.5
```

### 要求

- 使用逗号（`,`）作为分隔符
- 至少需要 4 列数据（ID、X、Y、Z）
- X、Y、Z 必须是有效的数字
- 编码建议：UTF-8 或 GB2312

### 示例文件

**coordinates.csv**
```
编号,东坐标,北坐标,高程
P1,1000.50,2000.30,100.50
P2,1010.50,2010.30,101.20
P3,1020.50,2020.30,102.50
P4,1030.50,2030.30,103.80
```

## 技术细节

### 工作流程

```
1. 加载 COM 库
    ↓
2. 用户选择导入模式
    ↓
3. 若需要，输入统一高度值
    ↓
4. 选择 CSV 文件
    ↓
5. 逐行读取文件
    ↓
6. 分割逗号分隔数据
    ↓
7. 验证坐标有效性
    ↓
8. 根据模式转换坐标
    ↓
9. 构建点列表
    ↓
10. 绘制多段线/3D多段线
    ↓
11. 恢复系统设置
    ↓
12. 显示结果信息
```

### 关键函数

| 函数 | 功能 |
|------|------|
| `split-str` | 按逗号分割字符串 |
| `atof` | 字符串转浮点数 |
| `read-line` | 逐行读取文件 |
| `command` | 执行 AutoCAD 命令 |

### 系统变量处理

脚本会临时修改以下系统变量以确保绘图准确：

| 变量 | 原因 |
|------|------|
| `CMDECHO` | 关闭命令回显，提高速度 |
| `OSMODE` | 禁用捕捉，防止坐标偏移 |

**执行完成后会自动恢复原始值。**

## 常见问题

### Q1: 文件打开失败

**原因：**
- 文件不存在或路径有误
- 文件被其他程序占用
- 编码格式不兼容

**解决：**
- 确认文件路径正确
- 关闭其他打开该文件的程序
- 尝试转换为 UTF-8 编码

### Q2: 坐标数据为空或不完整

**原因：**
- CSV 文件没有表头
- 数据列数不足 4 列
- 坐标值不是有效数字

**解决：**
- 添加表头行
- 检查列数是否符合要求
- 验证坐标数值有效性

### Q3: 绘制出来的多段线位置不对

**原因：**
- 坐标单位不匹配
- 坐标系设置不同

**解决：**
- 检查坐标数据的单位（米、毫米等）
- 确认当前 AutoCAD 坐标系设置

### Q4: 生成 2D 还是 3D 多段线？

**规则：**
- 模式 1（保持 3D）→ 生成 **3D 多段线**（`3DPOLY`）
- 模式 2（压平至 Z=0）→ 生成 **2D 多段线**（`PLINE`）
- 模式 3（统一高度）→ 生成 **2D 多段线**（`PLINE`）
